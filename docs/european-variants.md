# European Variants

The `cascade_europe_ifrs` scenario pack adds 9 European/IFRS/OECD variants of US-specific test cases. Each variant replaces US accounting standards, tax forms, and jurisdictional assumptions with their European equivalents while preserving the same underlying Cascade Industries data model.

## Quick Start

```bash
# Generate with EU variants (requires the accounting core as a dependency)
uv run python generate_test_suite.py \
    --packs cascade_accounting_core cascade_europe_ifrs \
    --output /tmp/test_suite

# Or generate everything
uv run python generate_test_suite.py --packs all --output /tmp/test_suite
```

EU variants are **not generated by default**. They must be explicitly selected via the `--packs` flag.

## Jurisdiction Profile

The Cascade Europe group is a fictional European subsidiary structure of Cascade Industries, Inc.:

| Entity | Jurisdiction | Legal Form | Functional Currency | Role |
|---|---|---|---|---|
| Cascade Europe Holding B.V. | Netherlands | B.V. | EUR | Group holding company |
| Cascade Pr&auml;zisionsteile GmbH | Germany | GmbH | EUR | Precision manufacturing |
| Cascade Mat&eacute;riaux Avanc&eacute;s SAS | France | SAS | EUR | Advanced materials / R&D hub |
| Cascade Distribution Services Ltd | United Kingdom | Ltd | GBP | Distribution (non-EU comparator) |

Key jurisdictional choices:

- **IFRS reporting** for all entities (the group prepares consolidated IFRS financial statements)
- **UK included as a non-EU European comparator** for post-Brexit VAT and import duty scenarios
- **Group revenue ~EUR 120M** -- below the EUR 750M Pillar Two GloBE threshold (this is an intentional judgment trap in TC-06-EU)
- **Fixed deterministic FX rates** (budget rate GBP/EUR 1.17) per IAS 21 translation rules

## Scoring

EU variants use the same 5-dimension scoring rubric (Correctness, Completeness, Format, Robustness, Communication) and the same 1-3 scale as US cases. The auto-grader handles EU variants via per-TC gold standards and rubric entries in `scoring/rubrics.yaml`.

Self-test for EU variants:

```bash
uv run python -m scoring.auto_grader \
    --self-test \
    --suite-dir /tmp/test_suite
```

This runs the self-test across all generated packs. EU gold standards must score 3/3/3/3/3 just like US cases.

## Design Decisions

### Why variants, not replacements

EU cases are additive -- they do not replace or modify the existing US cases. This preserves benchmark continuity: scores from before the EU pack was added remain directly comparable. The `--packs` flag controls which packs are generated; omitting `cascade_europe_ifrs` produces the same suite as before the EU pack existed.

### Why these 9 test cases

The 9 cases were selected because they have US-specific dependencies that would make them unrealistic or misleading for evaluating an agent operating in a European context. Test cases without strong jurisdictional anchoring (TC-01 bank reconciliation, TC-05 AR memo, TC-11 QoE, TC-13 forensic AP, TC-14 cash flow, TC-15 DCF, TC-17 deliverable assembly) work as-is for European scenarios and do not need variants.

### Europe is not one jurisdiction

There is no single European equivalent to many US forms and regimes. Each variant explicitly states which jurisdiction(s) it models and why. For example:
- R&D incentives (TC-08-EU) use France and Germany because they have the two largest R&D incentive programs in the EU
- VAT (TC-10-EU) exercises intra-EU supply chains plus a UK third-country import scenario
- Transfer pricing (TC-09-EU) uses OECD guidelines which apply across all EU member states

## Non-Goal Caveats

- The EU variants model benchmark scenarios for AI agent evaluation. They do **not** provide legal, tax, or accounting advice.
- Generated documents are not substitutes for professional judgment or compliance with any jurisdiction's requirements.
- The suite does not model every IFRS practical expedient, every member state's local GAAP overlay, or every national tax derogation.
- UK-specific post-Brexit scenarios are simplified to a single import VAT/customs duty framing.
